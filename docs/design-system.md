# Design System

## デザイン方針

**モダン × ダークベース**。装飾を抑え、情報密度と可読性を優先する。Linear / Vercel / Dovetail に近いトーン。

- 暗色基調（純黒に近いがわずかに灰）
- 余白を広く取り、罫線は極薄
- アクセントカラーは控えめ（淡い藤色）
- アニメーションは「現れる」「フェードする」など最小限
- 装飾的なネオン・グリッチ表現は採用しない

> 見た目は実装時に随時調整して構わない。本書は現時点（`ui-redesign` ブランチ）で導入されたトークンを基準としている。

## デザイントークン

`app/globals.css` の `@theme` で定義。Tailwind v4 のカスタムプロパティとしてそのまま参照できる。

### タイポグラフィ

- 基本は Tailwind デフォルト（`font-mono` を補助的に使用）
- 数字・時刻・ログ表示は `font-mono`
- セクションタイトル: `24px / weight 500 / letter-spacing -0.5px`
- モノラベル: `11px / uppercase / letter-spacing 1px / text-3` (`.mono-label`)

### 角丸 / 罫線

- カード: `border-radius: 12px`
- ボタン: `border-radius: 8px`
- プログレストラック: `border-radius: 3px`、高さ `3px`

### モーション

`globals.css` 定義のアニメーションを共通化:

| キーフレーム | 用途 |
|------------|------|
| `fadeUp` | スクロール出現 (`.animate-fade-up`) |
| `fadeIn` | フェードイン |
| `pulseGlow` | アクティブインジケーター |
| `fillBar` | プログレスバー初期表示 |
| `countIn` | 数値カウントアップ |

ステージ遅延クラス `.delay-1` 〜 `.delay-8` で順番に出現させる。

## 共通コンポーネントクラス

`globals.css` で定義済み:

| クラス | 用途 |
|-------|------|
| `.card` | カード（ホバーで持ち上がる + マウス追従グロー） |
| `.card-flat` | カード（ホバー昇降なし） |
| `.btn-primary` | 主要ボタン（白背景・黒文字） |
| `.btn-secondary` | 副次ボタン（半透明白） |
| `.section-title` | セクション見出し |
| `.mono-label` | 小ラベル（uppercase） |
| `.progress-track` / `.progress-fill` | プログレスバー |

## コンポーネント原則

- **新しいスタイルを書く前に、まず `globals.css` のトークンとクラスを使う**
- カラーは Tailwind のカスタム色（`bg-card`、`text-text-2` など）でも参照可
- マジックナンバー（生のカラーコード・px 値）を散らさない
- アクセントカラーは「強調すべき 1 箇所」だけに使い、画面に複数同時に出さない
- アイコンや絵文字を多用しない
- アニメーションは情報の理解を助けるものだけに留める

## やらないこと

- ネオングロー・グリッチ・スキャンライン等の装飾効果
- 多色配色（赤・緑・青・黄を画面内に同時に並べる）
- 派手な効果音・大きなトランジション
- カスタムフォントの追加（必要になるまで保留）
