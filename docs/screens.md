# Screen Specifications

各画面の責務・載せる要素・載せない要素を定義する。
コンポーネントや UI 実装はこの仕様を元に決定すること。

> 設計原則（参考: 一般的な personal life dashboard のベストプラクティス）
> - 最重要情報を最大・左上に置く（hierarchy）
> - 1 画面の情報量を抑え、詳細は対応するサブページへ
> - サブページは「カテゴリの全情報」を持つ（Overview とは情報の重複を最小化）
> - ウィジェット過多を避ける

## Overview (`/`)

### 役割
その日に **何をやるか / 何をやったか** を一目で把握する中核ハブ。
詳細はサブページに任せ、Overview は **タスク + 予定** に集中する。

### 載せるもの
1. **Hero**
   - 挨拶（Good morning, Kei 等）
   - **Year Progress ウィジェット**（今年の経過率を %・バーで表示）
2. **Today's Tasks** — GitHub Issues の未完了一覧
3. **Today's Schedule** — Google Calendar の当日予定

### 載せないもの
- 学業 / 家計 / 健康の詳細データ（→ 各サブページ）
- 行動ログのタイムライン（→ Health）
- 月次収支・固定費（→ Finance）
- パラメーター（HP/MP 等。ゲーミフィケーション要素は検討中）

### データソース
| 要素 | ソース |
|------|-------|
| Today's Tasks | GitHub Issues (`lib/github.ts`) |
| Today's Schedule | Google Calendar (`lib/google-calendar.ts`) |
| Year Progress | クライアント側で `Date` から計算（外部依存なし） |

### 1 画面表示
Overview は 1 画面（スクロールなし）に収めることを目指す。

## Academic (`/academic`)

### 役割
学業の進捗・予定・タスクを集約する。

### 載せるもの
1. **単位取得進捗** — 取得済み / 残り / 進捗率 / 卒業目標年
2. **科目区分別の進捗** — 総合教育 / 専門教育 / 卒業論文
3. **学業関連のタスク・予定**
   - GitHub Issues の `academic` ラベル付き
   - Google Calendar の学業関連イベント（カレンダー名やキーワードで判定）

### 載せないもの
- その他カテゴリのタスク・予定（→ Overview / 各サブページ）

### データソース
| 要素 | ソース |
|------|-------|
| 単位進捗 | `constants/academic.ts`（静的） |
| 学業タスク | GitHub Issues（label フィルタ） |
| 学業予定 | Google Calendar（カレンダー / キーワードで抽出） |

> 将来的に「学習時間ログ」を Webhook で記録することも検討（仕様確定後）。

## Finance (`/finance`)

### 役割
家計の状態を集約する。**個人情報のため外部 API 連携は基本行わない**。

### 載せるもの
1. **月次の収支サマリー** — 収入 / 支出 / 余剰
2. **今日の支出記録** — 当日の支出明細
3. **今後の大きな支出** — 近日中の予定支出（家賃更新等）
4. **固定費の内訳** — 毎月の固定費リスト

### 載せないもの
- その他カテゴリの情報

### データソース
| 要素 | ソース |
|------|-------|
| 月次収支 | iOS ショートカット → Webhook で日次入力 → Upstash KV に集計（**今後実装**） |
| 今日の支出 | 同上 |
| 今後の大きな支出 | 手動定義（KV または定数） |
| 固定費 | 手動定義（KV または定数） |

### プライバシー方針
- マネーフォワード等の外部家計簿サービスとの自動連携は **行わない**（個人情報リスク）
- 入力はすべて自分の手元（iOS ショートカット → Webhook）から
- KV に保存するデータも最小限にとどめる

> 当面はモックデータで UI を作り、データ入力経路は段階的に整備する。

## Health (`/health`)

### 役割
健康・生活リズムを記録・可視化する。

### 載せるもの
1. **今日の生活ログ** — 起床 / 就寝 / 運動 / 食事 / 外出 / 帰宅（既存 Webhook データ）
2. **週次・月次の習慣スコア** — 起床時刻のブレ、運動継続日数等
3. **睡眠リズムの推移** — 起床・就寝時刻のグラフ
4. **食事・運動の記録** — 回数・内容のメモ

### 載せないもの
- その他カテゴリの情報

### データソース
| 要素 | ソース |
|------|-------|
| 今日の生活ログ | Upstash KV `status:YYYY-MM-DD`（既存 Webhook） |
| 週次・月次スコア | 同上の集計（**今後実装**） |
| 睡眠リズム推移 | 同上の履歴データ（長期保存設計が必要） |

> 現状の KV キーは TTL 約 2 日。長期データを扱うには別キー設計が必要（→ `architecture.md` の未整理項目）。
