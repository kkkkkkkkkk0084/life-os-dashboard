# Gamification — 検討中

> **ステータス: 検討中**
> 採用するかどうかを含めて未確定。本書はコンセプトと検討項目のメモであり、確定仕様ではない。
> 実装に進む際は、別途確定仕様書を起こす。

## コンセプト

日々の活動（タスク完了 / 学業 / 健康行動 / 家計）を「自己管理を続けたくなる仕掛け」に変換する。RPG 的なメタファ（ステータス・経験値・実績）を借りる案があるが、見た目の路線（モダン × ダーク）と整合させる必要がある。

- ゴールは「ゲーム化そのもの」ではなく、**継続のためのフィードバックループを作ること**
- 派手な演出は採用しない（→ `design-system.md`）

## 検討中の要素

### 1. パラメーター（プレイヤーステータス）

候補:

| パラメーター | データソース案 |
|------------|--------------|
| 体力系（HP 相当） | 運動 / 食事 / 睡眠 |
| 精神力系（MP 相当） | 睡眠の規則性 |
| 知力系（INT 相当） | 取得単位 / 学習時間 |
| 資金（GOLD） | 月次収支 |

**未決事項**: 名称（HP/MP のような RPG 用語をそのまま使うか、もう少し中立的な名前にするか）、計算式の妥当性、表示頻度。

### 2. EXP / レベル

- 行動ごとに加点 → 累積でレベルアップ
- 数値で「積み上がっている感」を可視化したい
- **未決事項**: 必要か？ 単に行動ログだけで十分ではないか？

### 3. 実績（アチーブメント）

- 「7 日連続早起き」「単位◯個取得」のような達成バッジ
- **未決事項**: 解除条件の妥当性、通知方法、スコープ

### 4. 行動ログ（バトルログ）

- 当日の行動を時系列で表示（既にデータは Webhook 経由で記録されている）
- これは比較的採用しやすい（演出を抑えれば普通の Activity Log として成立する）

### 5. 週次レポート

- 1 週間の行動サマリーと前週比
- パラメーターの推移グラフ
- **未決事項**: 必要十分なメトリクス、配信方法（画面表示 / 通知）

## 検討すべき論点

1. **そもそもゲーミフィケーションが必要か？**
   単純な「行動ログ + 集計画面」で目的（継続）を達成できるなら、抽象レイヤーを増やす意味は薄い。
2. **モダン × ダークのトーンとどう両立させるか？**
   RPG 用語をそのまま使うとトーンと衝突する。比喩を抽象化（例: HP → Vitality、EXP → Activity Points）するか、用語自体を捨てるか。
3. **データの長期保存設計**
   現状は日次キー + TTL 2 日。レベル・実績・履歴を扱うなら別の保存戦略が必要。
4. **採用するなら段階的に**
   - フェーズ A: 行動ログ表示のみ
   - フェーズ B: パラメーター集計（数値のみ）
   - フェーズ C: EXP / レベル / 実績
